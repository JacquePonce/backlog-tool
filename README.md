# Personal backlog (YAML + board)

## Where to run commands

This folder is **not** in your home directory. On Jacqueline’s machine it is:

`~/Documents/cursor/troy-beta/backlog`

Use that path in Terminal (or `cd` there from Cursor’s integrated terminal, which starts in the workspace). Examples:

```bash
cd ~/Documents/cursor/troy-beta/backlog
.venv/bin/python scripts/render_board.py
.venv/bin/python scripts/serve_board.py
```

If you see `cd: no such file or directory: troy-beta/backlog`, you are in `~` — use the full path above.

## Files

| File | Role |

## Status values

Use exactly one of: `backlog`, `in_progress`, `blocked`, `done`.

## Refresh from Jira (after MCP / export)

1. Update [`ingest/last_jira_currentUser.json`](ingest/last_jira_currentUser.json) with a fresh `searchJiraIssuesUsingJql` result (`assignee = currentUser() ORDER BY updated DESC`, `maxResults` up to 100), or ask the agent to call the Atlassian MCP and save the JSON.
2. Run:

```bash
cd ~/Documents/cursor/troy-beta/backlog   # or: cd troy-beta/backlog from repo root
.venv/bin/python scripts/refresh_from_jira.py
```

That merges [`local_items.yaml`](local_items.yaml) with the Jira export, writes [`items.yaml`](items.yaml), and runs [`scripts/render_board.py`](scripts/render_board.py).

## Regenerate the board only (`BOARD.md` + `BOARD.html` + `board.html`)

```bash
cd ~/Documents/cursor/troy-beta/backlog
.venv/bin/python scripts/render_board.py
```

### View in your browser (recommended)

Run a tiny local server so the board opens as a normal webpage (not the in-app preview):

```bash
cd ~/Documents/cursor/troy-beta/backlog
.venv/bin/python scripts/serve_board.py
```

Your default browser should open **`http://127.0.0.1:8765/index.html`** (hub). If the port differs, use the URLs printed in the terminal. Change the port with `BACKLOG_PORT=9000`.

On the board: **New card** is top-right; each column has **▼ / ▶** to collapse or expand (saved for this tab in `sessionStorage`).

Do not rely on double-clicking HTML — use the local server so `board-data.json` loads (same as beta-timeline-style pages served over HTTP).

**Local “New card” tasks** live in `localStorage` under `backlog-board-custom-cards-v1`. They are **per browser origin**: use the **same host** every time (e.g. always `http://127.0.0.1:8765`, not sometimes `http://localhost:8765`) or the board will look empty after refresh. Avoid private/incognito if you need cards to persist.

First-time setup:

```bash
cd ~/Documents/cursor/troy-beta/backlog
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```
