## Status values

Use exactly one of: `backlog`, `in_progress`, `blocked`, `done`.

## Daily refresh (Jira + Slack + Calendar/Gemini + Confluence)

This is the recommended path once a day. Python does not call any MCP — the **agent** does the pulls, then you run one terminal command.

1. In Cursor, send the agent this prompt: **`daily backlog refresh`**. The agent follows [`ingest/DAILY_REFRESH.md`](ingest/DAILY_REFRESH.md) and writes five files under `ingest/`:
   - `last_jira_currentUser.json` — `assignee = currentUser() OR reporter = currentUser()` since the last watermark.
   - `last_slack_mentions.json` — `@` mentions in channels whose name contains `troy` / `us` / `da` / `ccf` (token-boundary match on `-`).
   - `last_slack_dms.json` — DMs and group DMs with unread activity.
   - `last_gcal_events.json` + `last_gemini_docs.json` — events you're an attendee of (even if you didn't attend) and action items pulled from the attached Gemini notes (EN and PT headings supported).
   - `last_confluence_comments.json` — comments on pages you authored or where you're mentioned, limited to comments that still need your reply.
2. In the terminal:

```bash
cd ~/Documents/cursor/troy-beta/backlog
make daily
```

That runs [`scripts/daily_refresh.py`](scripts/daily_refresh.py), which merges the five JSONs with [`local_items.yaml`](local_items.yaml), applies your overrides from [`local_overrides.yaml`](local_overrides.yaml), regenerates [`items.yaml`](items.yaml) + the board, and bumps [`ingest/sync_state.yaml`](ingest/sync_state.yaml). Re-runs are **incremental**: items keyed by stable ids (`jira-KEY`, `slack-<channel>-<ts>`, `gmeet-<eventId>-<idx>`, `cfl-<page>-<comment>`) are upserted, not duplicated.

To close a Slack / Gemini / Confluence loop (those sources have no native status), add the item id to `local_overrides.yaml`:

```yaml
overrides:
  slack-C07KD9QAV5L-1772641511.021559: { status: done }
  cfl-123456-789012: { status: done, note: "answered in comment" }
```

You can run a subset of sources: `.venv/bin/python scripts/daily_refresh.py --only jira,slack`.

## Refresh from Jira only (legacy)

If only the Jira export changed and you do not want to touch the other sources:

```bash
cd ~/Documents/cursor/troy-beta/backlog
make jira     # = .venv/bin/python scripts/refresh_from_jira.py
```

That uses the JQL in [`ingest/QUERY_PACKS.md`](ingest/QUERY_PACKS.md), merges [`local_items.yaml`](local_items.yaml) with the Jira export, writes [`items.yaml`](items.yaml), and runs [`scripts/render_board.py`](scripts/render_board.py). Issues mapped to **`front: other`** with **`status: done`** are skipped.

### View in your browser (recommended)

Run a tiny local server so the board opens as a normal webpage (not the in-app preview):

```bash
cd ~/Documents/cursor/troy-beta/backlog
.venv/bin/python scripts/serve_board.py
```

Your default browser should open **`http://127.0.0.1:8765/index.html`** (hub). If the port differs, use the URLs printed in the terminal. Change the port with `BACKLOG_PORT=9000`.

On the board: **New card** is top-right; each column has **▼ / ▶** to collapse or expand (saved for this tab in `sessionStorage`).

Do not rely on double-clicking HTML — use the local server so `board-data.json` loads (same as beta-timeline-style pages served over HTTP).

**Local “New card” tasks** live in `localStorage` under `backlog-board-custom-cards-v1`. They are **per browser origin**: use the **same host** every time (e.g. always `http://127.0.0.1:8765`, not sometimes `http://localhost:8765`) or the board will look empty after refresh. Avoid private/incognito if you need cards to persist.

First-time setup:

```bash
cd ~/Documents/cursor/troy-beta/backlog
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```
